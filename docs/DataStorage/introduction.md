# DataStorage

## cookies

- 存储在客户端，可以设置过期时间
- 每次请求都会携带在请求头中，可能会增加请求体大小
- 存储容量小，一般只有 4KB
- 可以设置作用域，可以设置在当前域名下有效，也可以设置在父域名下有效

## session 跟服务器端有关

- 存储在服务器端，不会每次请求都携带在请求头中
- 存储容量大，一般有 2MB
- 可以设置作用域，可以设置在当前域名下有效，也可以设置在父域名下有效
- session 是基于 cookie 的，session 的 id 会存储在 cookie 中
- session 的过期时间可以设置，也可以不设置，如果不设置，那么 session 就会一直存在，直到服务器关闭
- session 的存储方式可以是内存，也可以是数据库，也可以是文件

## localStorage

- 存储在客户端，不会每次请求都携带在请求头中
- 存储容量大，一般有 5MB
- 可以设置作用域，可以设置在当前域名下有效，也可以设置在父域名下有效
- 可以设置过期时间，也可以不设置，如果不设置，那么数据就会一直存在，直到浏览器关闭
- 可以存储字符串，也可以存储对象
- 可以通过 setItem、getItem、removeItem、clear 等方法来操作数据
- localStorage 的数据在浏览器关闭后依然存在，除非手动清除

## sessionStorage

- 存储在客户端，不会每次请求都携带在请求头中
- 存储容量大，一般有 5MB
- 可以设置作用域，可以设置在当前域名下有效，也可以设置在父域名下有效
- 可以设置过期时间，也可以不设置，如果不设置，那么数据就会一直存在，直到浏览器关闭
- 可以存储字符串，也可以存储对象
- 可以通过 setItem、getItem、removeItem、clear 等方法来操作数据
- sessionStorage 的数据只在当前页面有效，关闭页面后数据就会消失

## webSQL

- 存储在客户端，不会每次请求都携带在请求头中
- 存储容量大，一般有 5MB
- 可以设置作用域，可以设置在当前域名下有效，也可以设置在父域名下有效
- 可以设置过期时间，也可以不设置，如果不设置，那么数据就会一直存在，直到浏览器关闭
- 可以存储字符串，也可以存储对象
- 可以通过 openDatabase、transaction、executeSql 等方法来操作数据
- webSQL 的数据在浏览器关闭后依然存在，除非手动清除

## indexedDB

- 存储在客户端，不会每次请求都携带在请求头中
- 存储容量大，一般有 50MB
- 可以设置作用域，可以设置在当前域名下有效，也可以设置在父域名下有效
- 可以设置过期时间，也可以不设置，如果不设置，那么数据就会一直存在，直到浏览器关闭
- 可以存储字符串，也可以存储对象
- 可以通过 indexedDB.open、indexedDB.deleteDatabase 等方法来操作数据
- indexedDB 的数据在浏览器关闭后依然存在，除非手动清除
